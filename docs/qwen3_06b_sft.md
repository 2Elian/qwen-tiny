## 冷启动训到什么程度可以rl？

- 语言连贯，特殊标签输出稳定（使用/data1/nuist_llm/TrainLLM/attention-residuals-reproduction/data/eval/general_prompts_test.json，来测试稳定的think标签）
- 指令跟随的准确性（使用ifeval benchmark）
- 基本知识的准确性（比如：数学的一些定理，可以把收集你想RL领域的一批次事实数据集，比如数学定理等，来评估，至少不能比base model差，本文构建的数据为/data1/nuist_llm/TrainLLM/attention-residuals-reproduction/data/eval/knowledge_test.json）
- 推理的连续性（比如：推理链路较为完整）
- RL阶段要训的目标数据集，冷启动之后的准确性与base model的准确性对比（用deepmath数据来测）
- 保留足够的探索空间（在数据集上不能过拟合，不能让模型学会死记硬背，应该保留足够丰富的输出风格，即探索空间，评估使用同一题 k 次采样的多样性 ）
```text
  # 过拟合的模型（无探索空间）— 同一题 4 次采样：
  生成1: <think>\nThe limit is 0.\n</think>\n\boxed{0}
  生成2: <think>\nThe limit is 0.\n</think>\n\boxed{0}
  生成3: <think>\nThe limit is 0.\n</think>\n\boxed{0}
  生成4: <think>\nThe limit is 0.\n</think>\n\boxed{0}

  # 好的模型（有探索空间）— 同一题 4 次采样：
  生成1: <think>\nUsing binomial expansion...\n</think>\n\boxed{0}
  生成2: <think>\nLet me factor out x^{1/2}...\n</think>\n\boxed{0}
  生成3: <think>\nApply L'Hopital after substitution...\n</think>\n\boxed{0}
  生成4: <think>\nUse Taylor series for cube roots...\n</think>\n\boxed{0}
```

# 为什么zero3的grad_norm要比fsdp大？

```text
  ┌────────────────────┬───────────┬─────────────┐
  │        指标        │   Zero3   │    FSDP     │
  ├────────────────────┼───────────┼─────────────┤
  │ grad_norm 稳态范围 │ 1.4 ~ 2.0 │ 0.17 ~ 0.22 │
  ├────────────────────┼───────────┼─────────────┤
  │ 比值               │ ~8-10倍   │ —           │
  ├────────────────────┼───────────┼─────────────┤
  │ loss 曲线趋势      │ 一致      │ 一致        │
  └────────────────────┴───────────┴─────────────┘
```

  根因：梯度规范化（Gradient Normalization）的时机不同

  两个框架在报告 grad_norm 时，梯度所经历的缩放操作不同。

  FSDP（PyTorch）

  FSDP 的 grad_norm 是在梯度 all-reduce 之后计算的，也就是梯度已经完成了跨 GPU 的平均：

  每个 GPU 计算自己的 micro-batch 梯度: g_1, g_2, ..., g_N
    → all-reduce: sum(g_i)
    → 除以 world_size: avg = sum(g_i) / N
    → 如果还有 grad_accum_steps = G: avg = sum(g_i,j) / (N × G)
    → grad_norm = ||avg||

  ZeRO-3（DeepSpeed）

  ZeRO-3 的 grad_norm 是在梯度累积完成之后、但平均之前计算的：

  每个 GPU 计算自己的 micro-batch 梯度: g_1, g_2, ..., g_G (G 次累积)
    → 累加: accumulated = sum(g_j)
    → grad_norm = ||accumulated||    ← 没有除以 world_size

  数学公式

  FSDP  grad_norm ≈ ||(1/(N×G)) Σ g_i||
  Zero3 grad_norm ≈ ||Σ_local g_j||

  RATIO = Zero3 / FSDP ≈ N × G

  其中：
  - N = 数据并行的 GPU 数量（world_size）
  - G = 梯度累积步数（gradient_accumulation_steps）

  从你的数据来看比值约 8-10 倍，这意味着 N × G ≈ 8~10。常见组合：
  - 8 GPUs × 1 grad_accum_step = 8x
  - 4 GPUs × 2 grad_accum_steps = 8x

# 数据精度下降严重分析：                                                                                                                                                    
  问题 1：数据量太少 × 学习率太高 × Epoch 太多                                                                                                                                                  
                                                                    
    训练: 2 个 parquet 文件 → ~4500 条 → 2 GPU × 4 batch × 8 accum → 每 epoch 仅 69 step                                                                                                                                 
    3 epochs 只有 ~210 步，LR=2e-5 对 0.6B 模型偏高                                                                                                                                                                      
                                                                                                                                                                                                                        
    小数据 + 高 LR + 多 epoch = 严重过拟合。模型在 210 步内就记住了训练分布，丧失了泛化能力。                                                                                                                            
                                                                                                                                                                                                                       
  问题 2：纯数学数据导致格式能力退化                                                                                                                                                                                   
                                                                  
    base model 虽然没有被训过 <think> 格式，但它至少会偶尔输出 \boxed{}（48%）。SFT 后模型被"矫正"成必须输出 <think>，但训练数据中 \boxed{} 在 <think> 内部的结构，与 eval 时的 prompt                                   
    指令不完全对齐，导致模型混乱——不知道该不该写 \boxed{}。         
                                                                                                                                                                                                                       
  问题 3：生成长度翻倍的根源                                                                                                                                                                                           
    
    模型在过拟合后会"背诵"训练数据中的长推理风格（训练数据 mean token 1607），但它失去了对何时停下来的判断（训练数据中 \boxed{} 的位置是固定的），导致持续生成但不输出 \boxed{}。                                        
                                                                  
  ---                                                                                                                                                                                                          
  崩溃链路:                                                                                                                                                                                                            
    小数据(4500条) × 10 epochs × LR=2e-5                                                                                                                                                                               
      → 210 step/epoch × 10 epochs = 2100 次更新                                                                                                                                                                       
      → 模型严重过拟合训练数据的特定模式                                                                                                                                                                               
      → 丧失泛化能力（catastrophic forgetting）                                                                                                                                                                        
      → \boxed{} 输出率暴跌 (48% → 10%)                                                                                                                                                                                
      → 生成长度暴涨 (模型在"迷茫") (1336 → 2968)                                                                                                                                                                      
      → EM 暴跌 (28% → 4%)                                                                                                                                                                                             
                                                                                                                                                                                                                       
  base model 的 28% EM 说明它原本就会做题，只是不知道 <think> 格式。SFT 应该是在保持原有数学能力的基础上教会模型格式，但现在的设定把它原本的能力也摧毁了。                                                             
                                                                                                                                                                                                                       
  补救方案                                                                                                                                                                               
  ```text
           
                                                                  
  ┌──────────────────┬──────┬───────────────────┬────────────────────────┐                                                                                                                                             
  │       参数       │  旧  │        新         │          原因          │
  ├──────────────────┼──────┼───────────────────┼────────────────────────┤                                                                                                                                             
  │ num_files        │ 2    │ null (全部 10 个) │ 28K 样本减少过拟合     │
  ├──────────────────┼──────┼───────────────────┼────────────────────────┤                                                                                                                                             
  │ num_train_epochs │ 10   │ 1                 │ 数据多了，1 epoch 足够 │                                                                                                                                             
  ├──────────────────┼──────┼───────────────────┼────────────────────────┤                                                                                                                                             
  │ learning_rate    │ 2e-5 │ 1e-5              │ 0.6B 模型应更保守      │                                                                                                                                             
  ├──────────────────┼──────┼───────────────────┼────────────────────────┤                                                                                                                                             
  │ per_device_batch │ 2    │ 4                 │ 用满显存               │
  └──────────────────┴──────┴───────────────────┴────────────────────────┘      
  ```                                                                                                                                       
                                                                                                                                                                            
                                                                  
  ① 混入通用数据                                                                                                                                                                               
                                                                    
    冷启动 SFT 应该混合 ~80% 的通用指令数据，防止模型忘记基本的语言能力和格式输出。例如：                                                                                                                                
                                                                    
    训练数据 = 20% DeepMath-103K（文件1和2） + 80% 通用数据（如 OpenHermes, ShareGPT 等）                                                                                                                                            
                                                                                                                                                                                                                        
    这里是因为想让模型获得<think>能力，因为base模型的做题能力已经有40%了，在这里冷启动更多的数学数据没什么意义，数学数据集留给后续的RL来训，以获得更强的推理能力。所以，选择更多的通用数据集，先把稳定的<think>标签训出来。                                                                                                                                                                                                      
  ② 增加 warmup 步数                                                                                                                                                                                                   
                                                                    
    0.6B 模型在 28K 数据上 1 epoch 约 220 steps。warmup 应该覆盖前 10-15%：                                                                                                                                              
                                                                    
    warmup_ratio: 0.1   # 原来 0.05
